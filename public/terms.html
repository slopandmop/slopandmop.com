Terms

What this is

An experiment. A click on the front page provisions a temporary Intel TDX virtual machine on our GCP project and drops you into a chat with a Claude Code process running inside it. That chat can provision further VMs on the same project (for example, to run openclaw). Everything is torn down on a timer.

What's allowed

Exploring the ollama + openclaw demo. Asking your sm to deploy other slop from the catalog. Kicking the tires on confidential compute. One sm per source IP per day, soft cap.

What's not

Running anything illegal or abusive. Using the sm as a free general-purpose VM (it will be reaped). Attempting to exfiltrate the sm's credentials or attack other tenants.

Data

The markdown workspace inside your sm is persistent for the sm's lifetime only. When the watchdog tears it down, the workspace dies with it. The factory keeps a row with your sm_id, source IP, and timestamps until the sm is reaped plus 30 days.

No warranty

Nothing here is a production service. The sm might crash, the model pull might fail, the factory might return 500. Retry, and if it's still broken, file an issue.
